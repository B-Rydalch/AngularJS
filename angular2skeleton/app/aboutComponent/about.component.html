Intern Sandbox Documentation Policy

The commenting and documentation will go as follows for this sandbox.
Component status will be commented at the top of each 'ts' file. Status "completed", refers to no known bugs and is operational according to last edited date. In the event a component is changed in any way each intern is asked to state the status of the component. Place "completed" if componet is working as desired and update the last edited date. If component is under development, list any known bugs that is currenly being fixed or any new development taking place with the component. If component is under development, comment at the top of the 'ts' file and state what is under development and what is not working. This is to help all interns keep track of what the last intern was doing while performing tutorials. If an intern finds a good resource to use, we ask that you comment the sited source with a brief description and the url to that source in the "app.component.ts" file. Thank you for joining the Avercast team. We hope you enjoy your stay.
